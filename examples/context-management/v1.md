### dependency-preferences.md
**File:** `.claude/memory/dependency-preferences.md`
**When to use:** To remember user preferences for package management and dependencies
**References:** Works with dependency-awareness.md rule

**Content:**
```markdown
---
name: dependency-preferences
description: User and project preferences for dependency management
type: user
---

**Package Manager:** npm (default) or pnpm (if package-lock.json vs pnpm-lock.yaml detected)
**Dependency Strategy:** Prefer established, well-maintained packages with good TypeScript support

**Preferred Packages:**
- State Management: Context API (built-in) or Zustand for complex state
- HTTP Client: Built-in fetch or axios for complex scenarios
- Forms: React Hook Form (if form handling needed)
- UI Components: Tailwind CSS for styling (already configured)
- Date Handling: date-fns (lighter alternative to moment.js)

**Avoided Packages:**
- Avoid suggesting heavy frameworks when lighter alternatives exist
- Don't suggest packages that duplicate existing functionality
- Consider bundle size impact for client-side packages

**Version Strategy:**
- Use caret ranges (^) for most dependencies
- Pin exact versions for tools that affect build output
- Regularly update dependencies to maintain security
```

**Result:** Claude will suggest dependencies that align with established preferences
**Verify:** Ask for a date handling solution - Claude should suggest date-fns over moment.js
